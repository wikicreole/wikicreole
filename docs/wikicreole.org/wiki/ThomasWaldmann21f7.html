I would be happy if creole lives up to being a nice and useful common wiki markup.

But there is a long way to go and some people seem to rush it and do quick shots. This is not the way we will get something nice and useful.

Creole, if it wants to be a success, needs to address the use cases currently used wiki markup addresses. If it falls too short, it might be tiny, nice and common markup, but not useful due to being too limited.

Because of this, it is complete nonsense to list "creole support" on wikimatrix.org NOW (2006-09-07) or announce creole support for moin 1.6.

First let's get it sane and working, then reach for world domination - not vice versa.
